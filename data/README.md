# data/, the data contract + layout

ImageLab has two data contracts: ingestion (how an image enters the pipeline) and artifact (how a baked
representation reaches the web).

## Layout

| Path | What | Git |
|---|---|---|
| `raw/` | private/large source inputs | git-ignored (never committed; staged via `scripts/fetch-data`) |
| `images/` | the curated image set: procedural PNGs + a small licensed real subset + `index.json` (schema `imglab.imageset/v1`) | committed |
| `derived/_klt`, `_dict`, `_inr`, `_prim`, `_sym`, `_vae`, `_diff`, `_bench` | the compact per-representation artifacts the web replays | committed |

The procedural images are regenerated by `imglab.gen.synthetic` (CC0, license-clean by construction); the real
subset is fetched by `imglab.stages.fetch_images` under an explicit license allowlist.

## Contract 1, ingestion (image, pipeline)

Defined in `data-pipeline/imglab/io/image.py`. An image enters one of two ways:

- BUNDLE, a curated-set id resolved to `data/images/<id>.png`. Only ids whose `index.json` entry carries a
  license on the allowlist (CC0, CC-BY, public-domain, or an in-repo generator) are accepted; anything else is
  rejected, never silently served.
- RUNTIME, a user upload in the browser. It never touches the repo or the server: it is decoded to working
  planes in the page, run through the live tabs, and discarded on reload.

Every image is loaded to a fixed working size (256 px planes for the live tabs) and, for the curated set, a
hi-res variant (512 px) for the high-fidelity views. Loading is deterministic (fixed resample filter, fixed
size), so a baked reconstruction and its live twin operate on identical pixels.

## Contract 2, artifact (pipeline, web)

Each representation bake writes a compact index plus its arrays under `data/derived/<group>/`:

| Group | Index | Artifacts |
|---|---|---|
| `_klt` | (direct) | `patch.json`, the patch KLT/PCA basis |
| `_dict` | (direct) | `learned.json`, `overdct.json`, the sparse dictionaries |
| `_inr` | `index.json` (`trained[]`) | `<image>.json`, the trained SIREN weights |
| `_prim` | `index.json` (`fitted[]`) | `<image>.json`, the ordered primitive list |
| `_sym` | `index.json` (`fitted[]`) | `<image>.json`, the fitted closed-form equation (frequencies + coefficients) |
| `_vae` | `index.json` (`images[]`) | `<image>/NN.png`, the per-image VAE reconstruction + latent-perturbation frames |
| `_diff` | `index.json` (`images[]`) | `<image>/NN.png`, the per-image SD-Turbo image-to-image frames |
| `_bench` | (direct) | `index.json`, the measured cross-family benchmark |

`frontend/src/lib/contract.types.ts` mirrors these shapes so any drift fails the web build, and
`scripts/check_artifacts.py` verifies in CI that every index references files that exist and are non-empty. The
web loads only these committed artifacts, it never recomputes; the live tabs compute from the ingested image
directly (they read no derived artifact).

## Provenance / license

The procedural images are CC0 (generated in-repo). The real subset records its per-image source, license and
attribution in `images/index.json`. Only compact derived artifacts are committed; raw/private inputs stay out of
the repo (ADR-0055).
